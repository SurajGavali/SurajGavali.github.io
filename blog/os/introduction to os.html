Computer Organization

Modern Computer System

IMPORTANT

1. Processor(CPU) transfers data between itself and main memory(RAM) only!.

2. No data transfer between CPU and Hard-Disk, CPU and Keyboard, CPU and Mouse etc.

3. I/O devices transfer(How?) data to memory(RAM) and CPU instruction access data from the RAM.

How a modern computer system works?

First the data goes through the devices to memory then the CPU fetches the data from the memory and executes it!

A typical instruction–execution cycle, as executed on a system with a von Neumann architecture, first fetches an instruction from memory and stores that instruction in the instruction register. The instruction is then decoded and may cause operands to be fetched from memory and stored in some internal register. After the instruction on the operands has been executed, the result may be stored back in memory.

What does the processor do?

From the moment it's turned on until it's turned off, the processor simply does the following things

1. Fetch the instruction from the RAM(Memory)[Location is given by "Program Counter"(PC) register].

2. Decodes the instruction and execute it while doing this may fetch some data from the RAM.

3. While executing the instruction change/update the Program Counter.

4. CPU runs this steps in INFINITE LOOP i.e. Fetch,decode, change program counter, repeat(GO to 1).

FEW IMPORTANT TERMS

1. BIOS

The code "inbuilt" into your Hardware by manufactuerer it runs "automatically" when you start the computer and keep looking for a "Boot-Loader" to be loaded in RAM and to be executed

The Manufaturer stores this code where they keep "Program Counter(PG)"

2. Boot Loader

A program that exists on (typically sector-0 of) a secondary storage

It is loaded by BIOS in RAM and passed over control to "GRUB". It's jo is to locate the code of an OS kernel, load it in RAM and pass contri over it

3. Kernel

The code that is loaded and given control by BIOS initially when computer boots. It takes control of hardware(How? will see...)

Creates an environment for "application" to execute and controls access to hardware by applications etc.


Everything else is "Applications"

4. System Programs

Applications that depends heavily on the kernel and processor(E.g. Compiler.linker,loader)

Other Applications such as GUI, Terminal, Libreoffice, Firefox, VLC,...,Your own programs from Data structures etc.

Components Of a Computer System.

A computer system can be divided roughly into four components: the hardware, the operating system, the application programs, and the users.

The hardware—the central processing unit (CPU), the memory, and the input/output (I/O) devices—provides the basic computing resources for the system.

The application programs—such as word processors, spreadsheets, compilers, and Web browsers—define the ways in which these resources are used to solve users’ computing problems.

The operating system controls the hardware and coordinates its use among the various application programs for the various users.

An operating system is similar to a government. Like a government, it performs no useful function by itself. It simply provides an environment within which other programs can do useful work.
