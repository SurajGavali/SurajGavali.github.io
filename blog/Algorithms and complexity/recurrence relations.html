Recurrence Relations


• Many algorithms are recursive in nature. When we analyze them, we get a recurrence relation for time complexity.

• We get running time on an input of size n as a function of n and the running time on inputs of smaller sizes.

There are mainly three ways for solving recurrences.

1) Substitution Method :

• We make a guess for the solution and then we use mathematical induction to prove the guess is correct or incorrect.

For Example,

Consider the recurrence T(n) = 2T(n/2) + n

We guess the solution as T(n) = O(nLogn). Now we use induction to prove our guess.

We need to prove that T(n) <= cnLogn. We can assume that it is true for values smaller than n.

T( n ) = 2*T( n/2 ) + n

<= 2*(c)*(n/2)*Log(n/2) + n

= c*n*Logn - c*n*Log2 + n

= c*n*Logn - c*n + n

<= c*n*Logn

2) Recurrence Tree Method :

In this method, we draw a recurrence tree and calculate the time taken by every level of tree.

Finally, we sum the work done at all levels. To draw the recurrence tree, we start from the given recurrence and keep drawing till we find a pattern among levels. The pattern is typically a arithmetic or geometric series.

How to solve linear recurrence relationHomogeneous Recurrence Relations?

➔ Suppose, a two ordered linear recurrence relation is - AFn-1 + BFn-2

➔ The characteristic equation for the above recurrence relation is, x2 - Ax - B.

Three cases may occur while finding the roots −

• Case 1 :

If this equation factors as, ➔ (x - x1)(x - x2) and it produces two distinct real roots x1 , x2 then Fn = ax1n + bnx1n

• Case 2 :

If this equation factors as, ➔ (x - x1)2 then Fn = ax1n + bx2n

• Case 3 :

If the equation produces two distinct complex roots, x1 , x2 in polar form x1 = r∠θ and x2 = r∠-θ then following is the solution --> Fn = rn(acos(nθ) + bsin(nθ))

Example,

1. 5Fn-1 + 6Fn-2 , where F0 = 1 , F1 = 4

--> The characteristic equation of the recurrence relation is ➔ x2 - 5x + 6

➔ (x - 3)(x - 2)

➔ x1 = 2 , x2 = 3

--> Hence the solution is, Fn = ax1n + bx2n.

--> here, x1 = 2 and x2 = 3

--> Fn = a2n + b3n

--> F0 = 1 = a + b

--> F1 = 4 = 2a + 3b

Solving these two equations, we get a=2 and b=−1

Hence the final solution is, 2*2n - 1*3n.

Master Theorem

➔ This method is useful to solve the recurrences of the form, T(n) = aT(n/b) + f(n) where a ≥ 1 and b > 1.

➔ This recurrence describes an algorithm that divides a problem of size n into a subproblems, each of size n=b, and solves them recursively.

Consider,

T(n) = aT(n/b) + θ(nklogpn)

for a ≥ 1, b > 1, k ≥ 0 and p is a real number,

• Case 1 : if a > bk, then

➔ T(n) = θ(nlogba)

• Case 2 : if a = bk, then

➔ a. if p > -1, then T(n) = θ(nlogba logp+1n)

➔ b. if p = -1, then T(n) = θ(nlogba log(logn))

➔ c. if p < -1, then T(n) = θ(nlogba)

• Case 3 : if a < bk, then

➔ a. if p ≥ 0, then T(n) = θ(nk logpn)

➔ a. if p < -1, then T(n) = θ(nk)
