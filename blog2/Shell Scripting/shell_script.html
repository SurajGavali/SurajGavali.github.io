  • 2. Those who do it are way more productive than others
  • 3. Makes you think !
  • 4. Portable. Learn once, use everywhere on all Unixes, Linuxes, etc.

Path in UNIXES and LINUX systems

  • Tree like directory structure
  • Root directory is called/ represented by "/"
  • For example, "/home/student/a.c"
  • Every running program has a current working directory
    • "." (Dot) represents current working directory
    • ".." represents parent directory
    • Example, "./Desktop/xyz/../p.c" --> this represents "p.c" file in Desktop folder!

Some key concepts

  • Files don't open themselves!
    • Always some application/program "open()" a file
    • A file is displayed by the program which opens it. Each program has it's own way of handling files.

  • Programs don't run themselves
    • Running a program or clicking on a program is equivalent to requesting to OS to "RUN" it. The OS runs program!
    • Users (humans) request OS to run programs, using Graphical or Command line interface and programs open files
  • A command
    • Name of an executable file
    • For example: 'ls' is actually “/bin/ls”
    • Command takes arguments
      • E.g. ls /tmp/
    • Command takes options
      • E.g. ls -a
    • Command can take both arguments and options
      • E.g. ls -a /tmp/
    • Options and arguments are basically argv[] of the main() of that program
  • The Shell
    • Shell = Cover
    • Covers the OS's "System Calls" for hte application
    • Talks with Users and applications and does some talk with OS
    • Shell waits for user input and requests the OS to run a program which the user has asked to run.
    • Again waits for user's input.
