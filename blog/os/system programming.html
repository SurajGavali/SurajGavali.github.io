System Programming


What are compiler, assembler, linker & loader and C-library?

C-library

A compilation of some of the most frequently needed functions for c programs

Whenever we say C-library it is an machine/object code file containing the machine code of all these function

Where does the C-library o your computer?

---> /usr/lib/x86_64-linux-gnu-/libc-2.31.so

Compiler

It is a application program which converts one programming language to another most typically compliers convert high level language like C,C++ etc. to machine code language. E.g. GCC located at --> /usr/bin/gcc

Example : gcc main.c -o main --> here main.c is the c-code and main is object/machine code file.

Assembler

It is a application program that converts assembly code to machine code. assembly code is a human readable machine code language. E.g. x86 assembly. $ as something.s something

Compilation process using GCC for a C-program

gcc is just an another program(you can write it!). it takes C language file as input and does following things

0. Pre-processing : "#" directives get pre-processed.

E.g. #define MAX 10 (The preprocessor will replace all occurrences of "MAX" with "10" note that its a one zero not ten)

e.g. #include --> the contents of stdio.h will be copy pasted in this location. where is this file? /usr/include/stdio.h

If you run $ gcc -E main.c ---> this will run the pre-processor stage and stop.

1. Tokenization - split the files into tokens caled as lexical analysis

2. Symantic analysis - arranging the tokens into logical order according to grammer of the language and also identifying if there is wrong order. this is where you get most of your syntax errors

3. Converts the C-code(Tokenised and symmantically organised sequence of tokens) intoo assembly language.

If you want to see output of this stage then run $ gcc -S main.c

4. Invoke the assembler to convert assembly to machine language.(minus the linked code) , gcc -c main.c

5. invoke the linker to link the C-library and other libraries(if needed) with the machine code

Linking process

Before linking if you see the machine code of any c code file there will not be any code of called function it will just have call to that specific function.

when you run $gcc a.c b.c d.c -o try, then the machine code of try will have the code of called function link to that function.

What is ELF

Its full form is Executable and Linkable format.

When you run $ ./try , essentially there will be fork() and exec("./try","..")

So the OS has to open the file "./try" and understand it. So each OS will demand it's own object code file format. For example Windows supports "exe" format, while Linux supports "ELF" format.

ELF is used not only for executable(complete machine code) programs, but also for partially compiled files e.g. main.o and library files like libc.so.6

Loader

Is a "code" that loads the machine code from file to memory. E.g "exec" is loader(means loader is basically code of exec)

Loader is part of OS.

Cross compilation

Using a compiler on one platform(OS + processor) to generate machine code for another platform(target)

generally compilers by default will generate machine code for another platform on which they execute.

gcc is a good compiler can be used for cross-compilation.

Ex. you can run gcc + x86 to generate an executable file for ARM + some OS.

Cross compilation is needed if compiler does not exist on the "target" platform.

Compilers,linkers, assemblers etc. are system program because their behaviour and output depends on the system(system + OSs)

Static and dynamic linking

Static linking : The linker combines the machine code of all functions, into the executable. hence the executable has the large size.

Dynamic linking : The linker does not combines the code of all functions, typically the library functions into the executable it inserts "stub" code instead.

When the program starts running, at that time, the "stub" code will help the loader to locate the code of the library, and load it. thats why the size of a executable is small. and it provides the flexibility to change the library code without changing the executable.

Default is "Dynamic"

Static and Dynamic loading

This is job of a loader

If the entire program is loaded in memory at "exec" time then its static loading

if a particular function is loaded in memory when it's called, than its called as dyanamic loading

Now there arises 4 possibilities of Linking + Loading

1. Static Linking + Static Loading

2. Static Linking + Dynamic Loading

3. Dynamic Linking + Static Loading

4. Dynamic Linking + Dynamic Loading
